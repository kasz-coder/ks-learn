TeachFlow

How the AI-powered teaching system works — from chat to lessons, roadmaps, references, and records.

1 End-to-End Flow

A conversational AI tutoring system built on a chat-driven loop. The user types a message, the AI generates a response that may include structured ::: blocks, and those blocks are saved to the database as persistent learning artifacts.

💬User Message
→
🛡️Auth & Rate Limit
→
🧠Build Context
→
🤖AI (Agnes 2.0)
→
📡SSE Stream
→
💾Parse & Save Blocks
Key insight: The AI is instructed to output :::lesson[Title], :::reference[Title], :::record[Title], and :::roadmap[Title] blocks. The backend parses these with regex and upserts them into Supabase tables. The frontend renders them as clickable cards with iframe previews.

2 The System Prompt

The TEACH_SYSTEM_PROMPT (in lib/openrouter.ts:12) is a 119-line instruction that defines the AI teacher's personality and rules.

Core Philosophy

  • Knowledge — high-quality lessons (HTML with inline CSS)
  • Skills — interactive practice (quizzes with data-correct)
  • Wisdom — real-world application (tying back to user's "WHY")

Teaching Sequence

1
ASK

About the user's mission (WHY) if not yet defined

2
ASSESS

Gauge the user's current knowledge level

3
PLAN → roadmap block

Present a structured learning path as JSON with goal + steps

4
TEACH → lesson block

Create an HTML lesson with meta, insight, compare, quiz, and next sections

5
CHECK

Pose quiz questions to verify understanding

6
TRACK → record block

Log what was learned with a learning record

7
SUGGEST

Recommend next steps / further exploration

Key Rules

  • Only output plain text or ::: blocks — no XML, no JSON wrappers
  • Never use <style> blocks; use CSS class names only (meta, insight, compare, quiz, next, term, name, def)
  • Quiz options must use data-correct="true" or data-correct="false"
  • Keep lessons short — one concept each
  • Always tie back to the user's mission
  • Web search is available via Tavily API — cite sources with URLs
  • Language support: if user prefers non-English, the prompt appends a language instruction

3 Block Types

The AI expresses structured content via four block types. Each block starts with :::type[Title] and closes with :::.

lesson Lesson

:::lesson[Python Variables]
<div class="meta">Lesson · Basics</div>
<h1>Python Variables</h1>
<p>Variables store data...</p>
<div class="compare">
  <div class="bad">...</div>
  <div class="good">...</div>
</div>
<div class="quiz">
  <div class="q" data-correct="true">A</div>
</div>
<div class="next">...</div>
:::

reference Reference / Glossary

:::reference[Python Glossary]
<div class="term">
  <div class="name">Variable</div>
  <div class="def">A storage location...</div>
</div>
:::

roadmap Learning Roadmap

:::roadmap[Learning Path: Python]
{
  "goal": "Build a web app...",
  "steps": [
    { "order": 1, "title": "Basics", "status": "upcoming" }
  ]
}
:::

record Learning Record

:::record[Variables Session]
- Date: today
- Key insight: Variables are...
- Questions: What about constants?
:::

Fallback: Untitled Lesson

If no blocks are found but the response contains HTML-like tags (<div>, <h1>, etc.) and is >50 characters, parseBlocks() wraps it as an "Untitled Lesson" to avoid data loss (see lib/blocks.ts:71-76).

4 Chat API — POST /api/chat

The central endpoint in app/api/chat/route.ts. Here's exactly what happens:

1
Authenticate

Supabase SSR auth — rejects unauthenticated requests with 401

2
Rate Limit

In-memory per-user: 20 requests/minute. Returns 429 + Retry-After header if exceeded.

3
Validate Input

Zod schema: workspaceId (UUID) + message (1-10,000 chars)

4
Verify Workspace Ownership

Fetches workspace by ID + user_id — ensures RLS compliance

5
Save User Message

Inserts { workspace_id, role: 'user', content } into chat_messages

6
Check for User Roadmap

If the user typed a roadmap[...] block directly, upsert it to Supabase

7
Fetch History

Last 500 messages from chat_messages for this workspace

8
Fetch Existing Content

Parallel queries for lessons, references (5), learning records (10), roadmaps (1)

9
Summarize (if due)

Every 25 messages, call AI again to generate a 2-3 sentence summary. Store in workspaces.context_summary. This preserves long-term context when old messages drop out of the budget.

10
Build Context

buildContext() in lib/context.ts: system prompt + state block (topic, mission, lessons, references, records, roadmaps) + conversation summary + message history. Budget: ~24,000 chars (~6,857 tokens). Truncates oldest messages first.

11
Phase 1: AI Call (with tool support)

Calls Agnes 2.0 Flash with tools: [WEB_SEARCH_TOOL]. If the AI calls web_search, the query goes to Tavily API, results are injected as a tool response message, and Phase 2 streams the final answer.

12
Phase 2: SSE Streaming (or direct response)

If tool calls were made, stream the follow-up. Otherwise, use the direct response. Streams data: {"content": "..."} SSE events to the client.

13
Save Assistant Message

After stream completes, save the full response to chat_messages

14
Parse & Upsert Blocks

Fire-and-forget Promise.all(): parse ::: blocks, upsert to Supabase (lesson versioning, slug-dedup references, always-insert records, single-row roadmaps)

5 Token-Budgeted Context

lib/context.ts builds the AI's context window under a character budget:

Default Budget
24,000 characters (~6,857 tokens at 3.5 chars/token)
State Block
Shows topic, mission, and summaries of existing lessons/references/records/roadmaps (each truncated to 200 chars)
Conversation Summary
Optional second system message with a 2-3 sentence summary (generated every 25 messages)
History Truncation
Messages are walked in reverse (newest first). If within budget, included. If partially over budget but above 64-char threshold, truncated. Older messages dropped with a system note: "[N earlier message(s) omitted]"
Why token budgeting matters: Without it, long conversations would exceed the AI's context window. The summary mechanism ensures key information persists even as detailed chat history is pruned.

6 Database Schema (Supabase)

All tables have RLS enabled — users can only access their own data via workspace ownership.

Table Key Columns Unique / Notes
workspaces id, user_id, topic, mission, context_summary, summary_message_count One per learning topic. context_summary preserves long-term memory.
chat_messages id, workspace_id, role, content, created_at All conversation history. role = user | assistant
lessons id, workspace_id, title, slug, content, order, version Unique per (workspace_id, slug). Version auto-increments on update via upsert_lesson_with_version() RPC. Old versions saved to lesson_versions.
references_doc id, workspace_id, title, slug, content Unique per (workspace_id, slug). Slug-based dedup (update existing, insert new).
learning_records id, workspace_id, title, content Always inserts a new row — no dedup. Each record is an append-only log entry.
roadmaps id, workspace_id, title, goal, steps (jsonb), updated_at Single per workspace. If AI outputs a new roadmap, it updates the existing row.
lesson_versions id, lesson_id, version, title, content, created_at Version history for lessons. Browsable from the Resources page.
user_settings user_id, language Per-user language preference. Default: English.

7 Frontend Rendering

Chat Message (components/chat/chat-message.tsx)

The AI response is split by a regex into text parts and blocks:

  • Lesson / Reference — rendered as a clickable card with an <iframe> preview (scaled to 1/3 via CSS transform)
  • Learning Record — rendered as an inline purple-themed card showing the record content
  • Roadmap — rendered as a button linking to /workspaces/[id]/roadmap
  • Text — rendered in message bubbles with whitespace-pre-wrap

Full-Screen Lesson Viewer

When a user clicks a lesson or reference card, a full-screen overlay opens. The HTML is passed through:

  1. injectSharedStyles() — injects a comprehensive CSS stylesheet (from lib/shared-styles.ts) that styles .meta, .insight, .compare, .bad, .good, .quiz, .q, .next, .term, .name, .def, etc.
  2. injectQuizScript() — injects JavaScript that makes quiz questions interactive (click to reveal correct answer, green highlight + feedback)
  3. Rendered in a sandboxed <iframe> with srcDoc

Resource Gallery (app/(dashboard)/workspaces/[id]/resources/page.tsx)

Server-side rendered page showing all lessons, references, and learning records for a workspace. Filterable by type. Each card shows a type badge, title, and preview (iframe for lessons/references, bullet list for records).

Roadmap View (components/roadmap/roadmap-view.tsx)

Vertical timeline with numbered steps, progress bar, and toggle buttons. Clicking "Done" toggles completed ↔ upcoming via PATCH /api/workspaces/[id]/roadmap. Auto-refreshes every 5 seconds.

8 Client-Side Streaming

Stream Store (lib/chat-stream-store.ts)

An in-memory Map<workspaceId, StreamState> with pub/sub:

  • Survives navigations — streaming continues even if the user switches tabs
  • TTL eviction — stale entries (30 min no access) are cleaned up
  • State: streaming, streamingContent, error, sending, abortController, lastMessage

React Hook (hooks/use-chat-stream.ts)

Uses useSyncExternalStore to subscribe to the stream store from React components.

SSE Parsing (lib/chat-stream.ts)

sendChatMessage() posts to /api/chat, reads SSE data: lines, handles:

  • Rate limiting (429) with retry-after message
  • Safety filter responses (marked with "User Safety: safe/unsafe")
  • Empty responses — rolls back the user message
  • Network errors — preserves any partial content received
  • AbortController — allows cancellation mid-stream

9 Response Evaluation (lib/eval-response.ts)

An offline validation library (not called in the main chat flow) that checks AI output quality:

Lessons
Checks for <html>/<head>/<body> tags, recommended CSS classes (meta, insight, compare, quiz, next), <h1> heading, min content length
References
Checks HTML structure, term/name/def CSS classes, at least one term entry
Roadmaps
Validates JSON parseability, goal field, steps array, each step's title/status/order/description, ascending order check
Records
Minimum content length (10 chars), bullet list format (- or •)

Returns an EvalReport with passed boolean, issues[] (errors + warnings), and stats (block counts, text length).

11 End-to-End Example

Scenario: User creates a workspace "Python Basics" with mission "I want to build web apps" and asks "Create a roadmap for learning Python."
  1. Create workspace → POST /api/workspaces saves { topic: "Python Basics", mission: "I want to build web apps" }
  2. User sends "Create a roadmap for learning Python"
  3. API receives the message, saves user message, fetches context (no lessons/references/roadmaps yet)
  4. buildContext() produces: system prompt + TOPIC: Python Basics, MISSION: I want to build web apps, LESSONS CREATED: None yet
  5. AI generates a :::roadmap[Learning Path: Python Basics] with JSON steps
  6. Client streams the response as SSE chunks, renders text in real-time
  7. After stream: full response saved to chat_messages, parseBlocks() extracts roadmap JSON, upserts to roadmaps table
  8. User sees "View Roadmap: Learning Path: Python Basics" button in the chat
  9. User clicks → navigates to roadmap page, sees timeline with steps
  10. Next message: "Create a lesson about Python variables"
  11. AI generates :::lesson[Python Variables] with HTML: meta, h1, paragraphs, compare (bad vs good examples), quiz, next steps
  12. parseBlocks() extracts lesson, calls upsert_lesson_with_version() RPC to store in lessons with slug python-variables, version 1
  13. User clicks the lesson card → full-screen overlay with styled HTML + interactive quiz
  14. Later: If AI updates the same lesson, old version is saved to lesson_versions, version incremented
  15. After 25 messages: a summary is generated and stored on the workspace to preserve long-term context
